## 📖 About this application

You spend hours reading and writing code every day. Often, you lose track of the concepts you learn during these sessions. Aftercode changes this process. It monitors your coding activity, captures the logic behind your work, and turns your progress into personalized audio podcasts.

You listen to these summaries while you commute, walk, or relax. This creates a feedback loop that helps you retain knowledge. You focus on the code while the software handles the conversion.

The tool uses a Rust backend for speed and a React web interface to manage your library. It tracks your workflow through local SQLite databases, so your data stays on your machine.

## 🚀 Getting started

When you launch aftercode, the web interface opens in your default browser. This serves as your primary control panel.

1.  **Configure your settings:** In the settings menu, link your preferred AI coding tools. This allows the software to track your activity.
2.  **Start a session:** Click the Record button before you open your code editor.
3.  **Review your progress:** After you finish coding, click Stop. The engine processes your input.
4.  **Listen to your podcast:** The software generates an audio file. You find this in the library tab. Click Play to listen to your summary.

The system uses local storage for all activity logs. Your code remains private and does not leave your machine unless you authorize specific cloud features.

## 📂 Managing your library

The library tab displays a list of your past coding sessions. Each session shows the date, the duration, and the topics covered.

*   **Search sessions:** Type keywords in the search bar to find specific lessons.
*   **Export audio:** Click the download icon to save a podcast file as an MP3. This allows you to transfer lessons to your phone or portable media player.
*   **Delete records:** Remove old sessions to free up disk space.

## 🛠️ Troubleshooting common issues

If you encounter problems, check these items to resolve them:

*   **Application does not open:** Ensure you have an active internet connection during the first launch. The application needs to check for updates upon startup.
*   **Audio files sound robotic:** Go to settings and adjust the voice synthesis parameters. Higher quality settings use more data but provide clearer results.
*   **No activity captured:** Verify that your coding editor is running. The background process needs to see active changes to generate a meaningful summary.
*   **Disk space warnings:** The audio cache grows over time. Clear your cache once a month to prevent storage issues.

## 🛡️ Privacy and local data

Your data matters. Aftercode uses a local SQLite database to store your session history. No third-party servers monitor your code. The software connects to the API only to process the audio generation. You remain in control of every step of the process. If you want to clear your data, use the Wipe Data option in the settings menu. This action deletes all logs and local files permanently.

## 💡 Best practices for learning

To get the most value from your podcasts, follow these tips:

*   **Review daily:** Listen to your daily podcast within 24 hours of coding. This strengthens your memory of new concepts.
*   **Use tags:** Add tags to sessions that cover complex problems. This makes it easier to revisit difficult concepts later.
*   **Change playback speed:** Use the speed toggle in the player to catch details you missed the first time.
*   **Take notes:** If a podcast mentions a specific tool or library, pause the audio and document the name for further research later.
